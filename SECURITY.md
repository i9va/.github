# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any i9va repository, **do not open a public issue**. Please report it responsibly via email:

**opensource@i9va.systems**

Include as much detail as possible:

- The repository and component affected
- Steps to reproduce the issue
- Potential impact and severity
- Any suggested fix (optional)

## What to Expect

- **Acknowledgement** within 2 business days
- **Status update** within 5 business days
- We will work with you to understand and resolve the issue before any public disclosure
- Credit will be given to reporters who responsibly disclose vulnerabilities, unless anonymity is preferred

## Scope

This policy applies to all public repositories under the [i9va](https://github.com/i9va) GitHub organization.

## Supported Versions

Each repository maintains its own support policy. In general, only the latest stable release receives security fixes. Check the individual repository's documentation for specifics.
